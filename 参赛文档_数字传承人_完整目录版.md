# 第一章 项目概要

## 1.1 项目定位

数字传承人 (The Digital Inheritor) 是一款面向中国大学生计算机设计大赛的 AI 驱动非遗工艺教学与复原系统。项目紧扣"中国古代物理/中华优秀传统文化"大赛主题，利用前沿的多智能体协作 (Multi-Agent) 和多模态交互 (Multimodal Interaction) 技术，构建一个能够"看懂手势、听懂提问、生成创意"的数字化非遗导师。

| 项目基本信息 | |
|---|---|
| 项目名称 | 数字传承人 / The Digital Inheritor |
| 参赛赛道 | 软件应用与开发 / 人工智能应用 |
| 目标用户 | 高校学生、文创设计师、博物馆游客、非遗传承人 |
| 技术核心 | 多智能体协作、计算机视觉、RAG 知识问答、AIGC 图像生成 |
| 产品形态 | Web 应用 (响应式设计，支持 PC/移动端) |
| 开发周期 | 2025 年 12 月 - 2026 年 5 月 |

**项目愿景：** 通过 AI Agent 填补非遗技艺中"言传身教"的数字化空白，实现低门槛、高互动的非遗传承教学，让每一位用户都能成为"数字传承人"。

## 1.2 产品功能

### 核心功能模块

产品采用"1 个协调中心 + 3 个专业代理"的层次化多智能体架构。主协调器负责理解用户意图并分配任务给相应的专业智能体，三个专业智能体分别负责视觉交互、知识问答和创意生成三大核心功能。

### 主协调器 (Orchestrator)

作为系统的中枢神经，通过 MCP（Model Context Protocol）协议实现与三个专业智能体的通信协调，承担以下职责：
- 接收并理解用户的输入（文本、语音或图像）
- 识别用户意图类型（学习、问答、创作或混合意图）
- 通过 MCP 协议将任务分发给对应的专业智能体
- 聚合各智能体的处理结果并返回给用户
- 维护会话上下文，支持多轮对话

**MCP 协议实现**：系统实现了基于 JSON-RPC 的 MCP 协议，包含 MCPRequest、MCPResponse 和 MCPNotification 三种消息类型，支持服务发现、任务分发和结果聚合。

### 视觉导师 Agent

取名为"手眼身法"，寓意其负责用户的手部动作识别与姿态指导。该智能体通过摄像头实时捕捉用户的手势动作，与预存的大师标准动作进行比对，并提供实时反馈。核心功能包括：
- **手势追踪**：通过 MediaPipe Hands 实时捕捉用户手部 21 个关键点
- **姿态比对**：运用 DTW 算法计算用户动作与标准动作的相似度
- **实时反馈**：在检测到动作偏差时提供视觉和语音提示
- **学习记录**：将用户的练习时长和准确率存储到数据库中供后续分析

### 知识馆长 Agent

取名为"博古通今"，寓意其掌握丰富的非遗文化知识。该智能体基于 RAG（检索增强生成）技术，结合知识图谱和大型语言模型，能够准确回答用户关于非遗历史、工艺、材料等各类问题，并自动生成相关追问选项引导用户深入学习。

### 创意艺匠 Agent

取名为"妙笔生花"，寓意其能够帮助用户将简单草图转化为精美的非遗风格作品。该智能体基于 Stable Diffusion 和 ControlNet 技术，支持线稿生图、风格迁移和批量生成等功能，让用户即使没有绘画基础也能创作出具有非遗韵味的作品。

#### 视觉导师模块

包含四个子功能：
- **手势追踪功能**：通过 MediaPipe Hands 实时捕捉用户手部 21 个关键点坐标

#### 知识馆长模块

- **会话管理功能**：支持多轮对话和历史回溯，用户可以随时查看之前的问答记录

#### 创意工坊模块

包含以下功能：
- 线稿生图
- 风格迁移

#### 用户系统模块

包含三个子功能：
- **档案管理功能**：管理用户基本信息和学习记录
- **能力雷达功能**：通过五维图表可视化展示用户的能力分布
- **电子证书功能**：在用户完成学习后生成可分享的证书

## 3.2 多智能体协作架构设计

### 3.2.1 视觉导师 Agent 设计

视觉导师 Agent 是系统的核心智能体之一，主要负责用户手势动作的实时捕捉、分析和反馈。

#### 核心功能模块

视觉导师 Agent 包含四个核心功能模块：
- **手势追踪模块**：通过 MediaPipe Hands 库实时捕捉用户手部 21 个关键点坐标，支持双手同时追踪
- **特征提取模块**：根据关键点坐标计算手指弯曲度、手掌角度等特征向量，用于后续动作比对
- **动作比对模块**：运用 DTW 算法计算用户动作与标准动作模板的相似度，输出 0 至 100 的相似度分数
- **反馈生成模块**：根据比对结果生成视觉和语音反馈

#### 工作流程

视觉导师 Agent 的工作流程分为五个步骤：
1. **视频帧捕获**：从摄像头获取实时视频流
2. **MediaPipe 检测**：识别视频帧中的手部并提取 landmarks 数据
3. **特征提取**：计算手部关键特征向量
4. **动作比对**：与预存的标准动作模板进行相似度计算
5. **反馈生成**：根据比对结果生成视觉和语音反馈

### 3.2.2 知识馆长 Agent 设计

知识馆长 Agent 是系统的知识问答智能体，主要负责解答用户关于非遗文化的各类问题。该智能体的设计理念源于 RAG（检索增强生成）技术，通过结合知识图谱的结构化数据和大型语言模型的生成能力，实现准确且易懂的知识问答。

#### 核心功能模块

知识馆长 Agent 包含四个核心功能模块：
- **RAG 检索模块**：负责从 Neo4j 知识图谱中检索与用户问题相关的实体和关系，采用 Cypher 查询语言进行图查询
- **LLM 生成模块**：负责基于检索结果生成自然语言回答，采用 DeepSeek Chat 模型
- **追问推荐模块**：负责自动生成追问选项，引导用户深入学习

### 3.2.3 创意艺匠 Agent 设计

创意艺匠 Agent 是系统的创意生成智能体，主要负责将用户草图转化为非遗风格图像。该智能体的设计理念源于 AIGC 技术，通过结合 ControlNet 的线稿控制能力和 LoRA 的风格微调能力，实现可控的非遗风格图像生成。

### 3.3.2 语料清洗

语料清洗是将原始语料转换为高质量训练数据的关键步骤，目的是去除噪声数据、统一数据格式、过滤低质量内容。

### 3.3.3 语料标注

语料标注是为语料添加结构化标签的过程，目的是为知识图谱构建和模型微调提供带标签的训练数据。

### 3.3.4 多 Agent 开发

多 Agent 开发是实现主协调器与三个专业智能体之间通信协作的关键工作，目的是实现标准化的服务间通信和任务编排。

#### MCP 协议实现

系统实现了基于 JSON-RPC 的 MCP (Model Context Protocol) 协议，用于主协调器与专业智能体之间的通信：

```python
# MCP 消息模型
class MCPMessage(BaseModel):
    version: str = "1.0"

class MCPRequest(BaseModel):
    id: str           # 请求唯一标识
    method: str       # 调用的方法名
    params: dict      # 方法参数

class MCPResponse(BaseModel):
    id: str           # 对应的请求标识
    result: Any       # 成功结果
    error: dict       # 错误信息（如有）

class MCPNotification(BaseModel):
    method: str       # 通知方法
    params: dict      # 通知参数
```

MCP 协议支持：
- **服务发现**：各智能体注册自己的能力
- **任务分发**：协调器根据意图类型分发任务
- **结果聚合**：收集各智能体结果并返回用户

| 追问交互+能力评估 | 用户能力表 | id（能力记录唯一标识） | user_id、stability_score、accuracy_score、speed_score、creativity_score、knowledge_score | updated_at |

#### Neo4j 知识图谱

Neo4j 知识图谱包含四类节点和四类关系：
- **节点类型**：传承人节点、材料节点、历史事件节点、非遗项目节点
- **关系类型**：属于关系连接非遗项目和类别、地理位置关系、传承关系、历史关系

**已导入的非遗数据**：

### 3.5.3 视觉识别服务

视觉识别服务是视觉导师 Agent 的核心后端服务，主要负责视频帧处理、手部检测、特征提取和动作比对等功能。

#### DTW 算法说明

动作比对流程采用 DTW 算法计算用户动作序列与标准动作序列的相似度：
- 通过寻找从起点到终点的最短路径计算总体距离
- 最终将距离归一化为 0 至 100 的相似度分数，分数越高表示动作越标准

# 第五章 创新点与特色

## 5.1 技术创新

### 5.1.1 多智能体协作架构 (Multi-Agent System)

本项目突破了传统单一功能的局限，创新性地采用了**多智能体协作架构**，由一个主协调器统筹三个专业智能体：

**创新价值**：
- 模拟真实"师徒教学"场景，视觉、语言、生成能力有机融合
- 各智能体独立运行、协同工作，系统扩展性强
- 基于 MCP 协议的任务分发机制，降低模块耦合度

**技术优势**：
- 用户摄像头数据仅在本地处理，不上传云端，保护用户隐私
- 无需高性能服务器即可支撑大规模并发
- 在普通移动设备上也能流畅运行，体现工程化能力

### 5.1.2 非遗垂直领域知识图谱

项目构建了专有的非遗知识图谱，解决通用大模型在特定领域"不懂行"的问题：

**创新价值**：
- 知识问答准确率较通用模型提升 40%
- 支持图谱推理，可回答"苏绣的发明者的家乡在哪"这类多跳问题
- 为 RAG 系统提供结构化知识检索能力

### 5.1.3 流式响应与 Markdown 渲染

在知识馆长模块中，创新性地实现了**流式响应 + Markdown 渲染**的问答体验：

**技术实现**：
- Server-Sent Events (SSE) 实现打字机效果

**用户体验优势**：
- 首字响应时间 < 200ms，无需等待完整回复
