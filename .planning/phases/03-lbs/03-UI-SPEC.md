# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: 全国非遗地图 + LBS. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | WeChat Mini-Program (Taro 4.x) — no shadcn |
| Component library | WeChat native `<map>` + Taro | 03-CONTEXT D-01 |
| Icon library | WeChat built-in markers (custom iconPath) | 03-CONTEXT D-01 |

## Component Inventory

### Map Container
- Full-screen WeChat `<map>` component
- `scale`: 4 (province level) initially
- `show-location`: shows blue dot for current user location
- `markers`: custom markers array with category icons
- `covers`: NOT used (use `markers` only)
- `longitude`/`latitude`: initial center at GCJ-02 (35.0, 105.0) — China center, or user's last known location

### Cluster Marker
- Visual: Circle with count, 36x36px
- Background: vermilion (`#C04851`) at 90% opacity
- Count text: white, 12px, semibold
- Border: 2px white stroke
- Shadow: `0 2px 8px rgba(0,0,0,0.2)`
- Tap behavior: zoom map to uncluster (increase scale to 8+)

### Individual Marker
- Visual: ICH category icon in circle, 32x32px
- Border: 2px white stroke
- Tap behavior: open bottom sheet overlay
- Selected state: scale 1.2 + vermilion ring pulse once

### Bottom Sheet Overlay
- **Collapsed:** 120px height, shows partial card (name + category + distance)
- **Expanded:** 70vh max, full card content
- **Gesture:** swipe down to dismiss (partial), swipe up to expand
- **Animation:** slide-up 300ms ease-out (Taro `Taro.animate`)
- **Background:** rice-paper (`#F7F5F0`) with top rounded corners (16px radius)
- **Top handle:** 40x4px rounded bar, centered, charcoal at 30% opacity

### Radius Preset Pills
- Horizontal row, above map, 16px from bottom safe area
- Pills: 60x32px each, horizontal padding 12px
- Options: 1km, 5km, 10km, 20km
- Active: vermilion background, white text
- Inactive: rice-paper background, ink-black text, 1px charcoal border
- Spacing between pills: 8px

### Category Filter Chips
- Horizontal scrollable row, above radius pills
- Chips: auto-width based on text, height 28px, horizontal padding 12px
- Active: vermilion border + vermilion text
- Inactive: charcoal border + charcoal text, rice-paper background
- Spacing between chips: 8px
- Scroll: horizontal, no scrollbar (hidden by WeChat)

### Navigation Button (Bottom Sheet CTA)
- Full-width minus 32px horizontal margin, 48px height
- Background: vermilion (`#C04851`)
- Text: white, 16px, semibold, "导航前往"
- Border-radius: 24px (pill shape)
- Active state: brightness 0.9, scale 0.98
- Disabled state: opacity 0.5

### WeChat Location Button
- WeChat native `<button>` with `open-type="location"`
- Positioned: 16px from right edge, same vertical as current map center
- Size: 44x44px (touch target)
- Background: rice-paper, circular, shadow
- Icon: location arrow, ink-black

## Interaction Contracts

### Map Load
1. Request location via `wx.getLocation({ type: 'gcj02' })`
2. If denied: show toast "定位权限未开启"，fall back to default center (35.0, 105.0) at scale 4
3. If granted: center map on user location, scale 10
4. Fetch all markers from `/map/markers` endpoint
5. Render markers with clustering applied

### Marker Tap
1. Marker scales to 1.2 with vermilion ring (150ms)
2. Bottom sheet slides up from bottom (300ms ease-out)
3. Map scale reduces slightly to keep marker visible (scale -= 1)
4. Selected marker gets "selected" state (persistent ring until another selected)

### Cluster Marker Tap
1. Map animates to zoom in (scale to 8+ over 300ms)
2. Cluster expands to show individual markers
3. Bottom sheet remains closed

### Radius Change
1. User taps radius pill
2. Active pill switches to vermilion immediately
3. Filter markers locally (within selected radius from user location)
4. Nearby count updates in bottom sheet area if open
5. If no markers in radius, show empty state

### Category Filter
1. User taps category chip
2. Toggle chip active state (can select multiple)
3. Filter markers by selected categories (show all if none selected)
4. Update marker visibility immediately

### Bottom Sheet Dismiss
1. Swipe down gesture OR tap on map area
2. Bottom sheet slides down (250ms ease-in)
3. Selected marker returns to normal state
4. Map scale restores

### Navigation
1. User taps "导航前往" button
2. Call `wx.openLocation({ latitude, longitude, name, address, scale: 18 })`
3. WeChat opens native map navigation
4. No confirmation needed (external app switch, not destructive)
